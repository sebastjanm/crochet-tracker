## Key Library Choices

### Why Expo?
- **Cross-platform**: Single codebase for iOS, Android, and Web
- **Rich APIs**: Pre-built native modules (camera, location, etc.)
- **EAS**: Simplified build and deployment
- **Active Development**: Regular updates and great documentation

### Why Zustand over Redux?
- **Simplicity**: Less boilerplate
- **Performance**: Minimal re-renders
- **TypeScript**: Excellent type inference
- **Size**: Smaller bundle size (~1KB)

### Why Lucide Icons?
- **Comprehensive**: 1000+ icons
- **Consistent**: Uniform design language
- **Customizable**: Easy to style and size
- **Tree-shakable**: Import only needed icons

### Why NativeWind?
- **Familiar**: Tailwind CSS syntax
- **Fast**: Compile-time styles
- **Type-safe**: TypeScript support
- **Platform-aware**: Adapts to native platforms

### Why AsyncStorage?
- **Native**: Built for React Native
- **Simple**: Key-value storage
- **Offline-first**: No server dependency
- **Fast**: Asynchronous operations

### Why Google Generative AI?
- **Multimodal**: Text, images, and chat
- **Fast**: Low latency responses
- **Flexible**: Multiple model options
- **Cost-effective**: Competitive pricing

## Bundle Size Impact

### Largest Dependencies
1. `react-native` (~25MB)
2. `expo` (~15MB)
3. `@tanstack/react-query` (~200KB)
4. `zustand` (~1KB)

### Optimizations
- Tree-shaking enabled for icons
- Code splitting via Expo Router
- Image optimization with Expo Image
- Lazy loading of AI features

## Update Strategy

### Regular Updates
- Expo SDK (major versions quarterly)
- Security patches (as released)
- React Native (following Expo compatibility)

### Version Pinning
- All dependencies pinned to exact versions
- Lock file (`bun.lock`) committed to repo
- Tested upgrades on separate branches
